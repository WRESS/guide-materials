フォルダ構造の説明

概要

このページではバックアップと同期によって同期したファイル・フォルダがそれぞれどのような意味を持つかを説明します。

フォルダ構造

基本的なフォルダの構造は次のようになっています。

  • guide-materials
    • assets
    • content
      • asakusa
      • common
      • info
      • kokyo
      • meiji-jingu
      • （その他）
    • guide-materials
    • layouts
    • resources
    • static
    • config.toml

assetsフォルダ

CSSとJavascriptに関するファイルが入っています。 もしサイトのデザインをいじりたくなったら後述するlayoutsフォルダ内のファイルと合わせて編集してください。

assetsフォルダ内の.cssファイルや.jsファイルはビルド時に改行や不要な記述が削除されてWebサイト向けに最適化されます。

（CSSに関してはSCSSでもっと見やすく書き直す計画があったのですが、時間が足りませんでしたごめんね！）

contentフォルダ

一番重要なフォルダです。 contentフォルダ内のファイルがビルド後に各記事になります。

HugoにはSectionという概念があり、Sectionの下に各記事が所属するという構造になっています。

Sectionはフォルダがあると生成されます。 つまり、contentフォルダ以下の各フォルダが各Sectionと対応しています。 対応関係は次の表のとおりです。

フォルダ名Webサイト上での表記
info基本情報
common共通知識
meiji-jingu明治神宮
kokyo皇居東御苑
asakusa浅草寺

なお、これ以外にもフォルダはありますが、基本的にはいじらなくて大丈夫です。 一応次のような役割があります。

フォルダ名役割
admin引継ぎ用の記事がここに入っています。この記事もここに入っています。
kyoto2019京都ガイド2019のしおりを作ったときのファイルがここに入っています。
offlinePWA化後、オフライン時にアクセスしたときに表示するページの基本設定をするためのファイルが入っています。
print「基本情報 > 印刷」ページからアクセスできる印刷用ページに関する基本設定を記述しています。
searchメニューバーにある検索に関する基本ページを生成しています。

詳しい編集の仕方については編集者用のページで行います。

guide-materialsフォルダ

Hugoでビルドを実行すると、完成したファイルがこのフォルダに入ります。 Github DesktopはこのフォルダをGithub上のリポジトリと同期してくれますが、そのGithub上のリポジトリがWebサイトの本体となっています。

layouts

Webサイトの基本となるHTMLファイルの素材群が入っています。 構造については少し複雑です。Hugoの根本的な仕様から解説しなくてはならなくなるため、ここでは省略します。

Guide Materialsの構造を大幅に変えたくなったときは、例えば次のようなサイトを参考にして、Hugoの仕組みについて少し勉強してみてください！ Webサイト作りに役立つし面白いですよ！

日本語のドキュメントも充実しているので、困ったら検索してみてください！

  • Hugo Documentation（英語、Hugo公式ドキュメント）
  • まくまくHugo/Goノート

ついでにここに書いておきますが、このサイトではCSSフレームワークとしてBootstrap Material DesignというBootstrapのフォークを利用しています。 Bootstrapは日本語のドキュメントも充実しているので調べてみると面白いと思います。

resources

Hugoビルド時のキャッシュファイルが入る場所です。 もしビルドしても画像がおかしかったりなど、挙動がおかしい場合はこのフォルダを削除してからビルドかけてみると良いと思います。

static

ロゴなどウェブサイト全体で使う画像ファイルがここに入っています。 なお、各記事に使う画像ファイルは編集者用の記事でも記述しますが、content内の各記事のフォルダに入れることになります。ここに入れないでください。

config.toml

Hugoの設定ファイルです。基本的にいじらなくて大丈夫です。 もし必要であればHugo Documentation（英語、Hugo公式ドキュメント）辺りを参考にしながらいじってい見てください。

最後に

上述の通り、記事を追加・編集するだけなら、contentフォルダ以下をいじるだけで大丈夫です。

もし大幅にサイトの構造を変える場合はHugoの仕組み、HTML、CSS、Javascriptについてある程度理解する必要があります。 あまり大幅な構造の変更でなければ頼まれればある程度は僕も対応するので、自力での編集が厳しければ連絡ください。
