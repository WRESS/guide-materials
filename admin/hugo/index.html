環境の構築（Hugo）

Hugoの導入

このページではマテリアルの素材ファイル群から実際にWebサイトに表示するHTML、CSS、Javascriptのファイルや処理済みの画像ファイルなどを生成するためのソフト、Hugoを導入します。

しかし、Hugoにはインストーラがなく、自動更新機能などが備わっていないソフトなので、本来は、パスを自動で通さなければならなかったり、定期的に更新しないといけません。 ここではそれを補うためにパッケージ管理ソフトを導入して、簡単に導入・更新できるようにしてHugoを導入します。

パッケージ管理ソフトの導入

Windowsの場合

Chocolateyというパッケージ管理ソフトを導入します。

まずはコマンドプロンプトを管理者権限で実行します。 次の画像のように、Windowsの検索欄に「powershell」と入力するとPowerShellが現れます。「管理者として実行」をクリックしてください。

実行すると青い（黒い）画面が出てきます。ここにコマンドを入力してインストールします。 コマンドは公式サイトに書いてあります。 2019年9月時点では次の通りです。

Set-ExecutionPolicy Bypass -Scope Process -Force; iex ((New-Object System.Net.WebClient).DownloadString(‘https://chocolatey.org/install.ps1'))

このコマンドをコピーしてPowerShellに貼りつけ、エンターキーを押して実行、あとは指示に従ってインストールしてください。

インストールが完了したらPowerShellに「choco」と入力してみてください。インストールされているChocolateyのバージョンが表示されれば完了です。

Macの場合

Homebrewというパッケージ管理ソフトをインストールします。

公式サイトに詳しいインストール方法が記載されていますので、そちらに従ったください。

ざっくり説明すると、macの「ターミナル」を起動して、コマンドを公式サイトからコピーし、ターミナルに貼りつけすればOKです。

Hugoのインストール

いよいよHugoをインストールします。

Windowsの場合

PowerShellに次のコマンドを入力します。

choco install hugo -confirm

これでインストール完了です。

macの場合

ターミナルに次のコマンドを入力します。

brew install hugo

これでインストール完了です。

バッチファイルの作成

Hugoを実行するときに一緒に実行する命令を記述したバッチファイルがあるとダブルクリックでHugoを実行できて便利です。 ここではバッチファイルを作成していきます。

Windowsの場合

次の「その１」と「その２」へのリンクを右クリックして「名前を付けて保存」を選択、デスクトップへ保存してください。

  • その１
  • その２

これらの詳しい使い方は以降の記事で説明しますが、ここで概要をさらっと解説します。

その１から保存できるmaterials.batはマテリアルをビルド（マテリアルの素材ファイルをWebサイトへ対応するファイルへ変換）するのに使います。起動すると「guide-materials」フォルダ内の「guide-materials」フォルダに生成物を生成してくれます。

その２から保存できるmaterials-test.batはマテリアルの表示確認をするのに使えます。起動すると数秒で表示確認用サーバーを立ち上げてくれます。 「 http://localhost:1313/guide-materials/ 」（materials-test.batを起動すると自動で開きます） にアクセスすると現在あなたのPCに保存されているマテリアルの素材ファイルから構成したWebサイトのプレビューが見られます。このプレビューを見ながら記事を編集すると便利です。

macの場合

バッチファイルを生成する方法がわかりません。 ターミナルからhugoとタイプするとHugoが実行できます。

詳しくはHugo公式サイトをご覧ください。
