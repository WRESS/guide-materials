セットアップ

記事の編集

前の記事で確認したように，各フォルダに存在している「index.md」ファイルを編集します。

記事を編集するにはGoogleドライブ上でそのまま編集するか，Googleドライブ上のファイルをPC上に同期して，プレビューしながら編集するかの2パターンがあります。

前者は何もインストールする必要がなく，楽ですが，編集した内容が正しく表示されるかどうかは管理者がビルドしてくれるまでわかりません。

後者は最初こそ複数ソフトをインストールする必要があるので少しだけ大変ですが，それ以降は快適に編集できます。

個人的には可能である限り後者がおススメです。編集者・管理者間で取り決めがある場合にはそちらに従ってください。 これから2つの方法のセットアップ方法についてそれぞれ記述していきます。

Googleドライブ上でそのまま編集する場合

Googleドライブ上にアプリを追加することにより、ソフトのインストールなしで，ブラウザ上でもMarkdownファイルの作成・編集が行えます。 ここでは、こちらの方法を解説します。

まずはGoogleドライブにアプリを追加する方法を説明します。Googleドライブを開き、何もない場所で右クリックした後、「その他」にマウスカーソルを合わせてください。すると新たなメニューが出てくるはずなので、一番下まで移動して、「アプリを追加」を選択してください。

アプリを追加ウィンドウが出てくると思うので、右上の検索窓に「Markdown」と入力してください。すると、「StackEdit」というものが一番上に出てくるはずです。今回はこれを使います。右の「接続」ボタンを押しましょう。

数クリックで追加が完了します。そのままOKを押します。

これで、Googleドライブ上で、すべての拡張子が.mdのファイルをダブルクリックするだけで開けるほか、右クリックメニューからStackEditを選択することで、Markdownファイルを作成することも可能になりました。

StackEditでは、左側にMarkdown形式で記入すると、右側に見出しが大きくなったり、リンクが貼られたりしたプレビューが表示されるようになっています。

GoogleドライブをPCに同期する場合

次のソフトをインストールします。

  • Google Drive（バックアップと同期）
  • Chocolatey or Homebrew
  • Hugo
  • Visual Studio Codeなどのコードエディタ

コードエディタ以外のインストール方法は次の管理者向けの各記事を参照してください。

  • 環境の構築（ドライブ）
  • 環境の構築（Hugo）

また，Markdownを編集する際はコードエディタが便利です。大学の講義などの関係でAtomやVisual Studio Codeなどがインストールされている場合はそれを使ってください。

インストールされていなければインストールしましょう。AtomかVisual Studio Codeがおススメです。

各コードエディタで，「ファイル > フォルダを開く」などからまるまるマテリアルのフォルダを開いておくと編集が楽です。
