仕組み

マテリアルの仕組み

このページではこのマテリアルがどのように管理されるかを説明します。大雑把には次の図の通りに処理されています。

Markdown形式で記事を書く

Markdownという形式で記事を書いています。 Markdownというのはある程度簡単かつ直感的にHTMLを書くための書式です。

例えばこの文章は次のような感じで書いています。

## Markdown形式で記事を書く
Markdownという形式で記事を書いています。
Markdownというのはある程度簡単かつ直感的にHTMLを書くための書式です。

例えば次のような感じで記事が書けます。

Markdownを用いた記事の書き方に関しての詳細は「記事の追加」のページをご覧ください。

Google Driveで管理する

このマテリアルの元ファイルはすべてGoogle Drive上で共有・管理しています。 これにより複数人で同時にマテリアルを管理することができます。

このマテリアルの元ファイルは全て 「WRESS2019ルートフォルダ > Guide > 新マテリアル > guide-materials」 以下に保存されています。

HugoでHTMLを生成する

出来上がったMarkdownの記事をHTML（Webページを記述するためのファイル）に変換したり，全ての記事に同じヘッダー・フッターを付けたり，画像を自動で適正なサイズに変換したりする処理をHugoというフリーソフトで行います。

Githubにアップロードする

Hugoによって生成したWebサイト用のHTMLや画像などのファイル群をGithubにアップロード（comitしてfetch）します。 Github Pagesというサービスにより，Webサイトとして公開できます。

役割分担について

HugoでHTMLを生成したり，Github上に生成したファイルをアップロードしたりするには，専用のソフトをインストールせねばならず，若干面倒です。そのため，2019年度は僕のPC上にのみHugoとGithubに関するソフトをインストールし，あやなんには記事をMarkdownで書いて，Google Drive上に上げてもらうまでをやってもらっていました。

このような分業が可能なため，以降あやなんのように記事を書いてGoogle Driveにあげる人を編集者，はやとのようにHugoでビルドしてGithubで公開までする人を管理者と呼びます。

管理者は最低一人いれば問題ありません。むしろ全員管理者でも問題ないです。

以降の記事は管理者向けと編集者向けに分けて記述します。
